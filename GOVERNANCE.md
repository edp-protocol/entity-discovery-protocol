# Governance

## Principles

1. **Open Development**: The spec is developed in the open
2. **Pragmatism**: Real-world usability over theoretical purity
3. **Backward Compatibility**: Breaking changes require major version bumps

## Decision Making

### Minor Changes
Documentation fixes and clarifications can be merged by any maintainer.

### Spec Changes
Changes to SPEC.md or schemas require:
1. An open issue for discussion (minimum 7 days)
2. Consensus among maintainers
3. PR review and approval

### Breaking Changes
Require:
1. Extended discussion period (minimum 30 days)
2. Explicit approval from all maintainers
3. Major version bump

## Versioning

EDP follows Semantic Versioning:
- **MAJOR**: Breaking changes
- **MINOR**: New features, backward compatible
- **PATCH**: Bug fixes, clarifications

Current version: **0.1.0 (Draft)**
